# 안내 메일 발송 기능 Design Spec

## Goal

구글시트 CRM에서 S열 체크박스를 통해 발송 대기를 표시하고, 커스텀 메뉴 버튼으로 담당자가 직접 검토 후 안내 이메일을 발송하는 기능.

---

## Architecture

Apps Script `onEdit` 트리거가 S열 체크박스 변경을 감지해 시각적 대기 상태를 표시하고, 커스텀 메뉴 "안내 메일 발송"을 통해 현재 로그인한 계정으로 GmailApp을 사용해 발송한다. 각 담당자가 자신의 계정으로 실행하므로 from 주소가 자동으로 본인 계정이 된다.

---

## 컬럼 참조

| 열 | 인덱스 (0-based, B=0) | 내용 |
|----|----------------------|------|
| G  | 5 | 수신자 이메일 |
| L  | 10 | 담당자명 (발신자 서명 치환) |
| N  | 12 | 월 발권량 (템플릿 분기 기준) |
| S  | 17 | 안내 메일 발송 일시 (체크박스 → 날짜) |

---

## 사용자 플로우

1. 담당자가 담당자명, 전화번호, 이메일, 주소 등 행 데이터 입력 완료
2. S열 체크박스 체크 → 해당 행 배경색 노란색으로 변경 (발송 대기 표시)
3. 담당자가 내용 검토
4. 상단 커스텀 메뉴 **"📧 안내 메일 발송"** 클릭
5. 확인 다이얼로그 표시 (예: "3건의 안내 메일을 발송하시겠습니까?")
6. 확인 클릭 → 이메일 발송
7. S열 체크박스 → `yyyy-MM-dd` 텍스트로 교체, 배경색 초기화

## 첨부파일

- Script Properties 키: `ATTACHMENT_FILE_ID`
- 값: 구글 드라이브 파일 ID
- 매달 드라이브에 새 파일 업로드 후 ID만 교체

---

## 에러 처리

| 상황 | 처리 |
|------|------|
| G열 이메일 비어있음 | 해당 행 스킵, 발송 완료 후 스킵된 행 목록 알림 |
| S열이 이미 날짜 텍스트 | 재발송 스킵 |
| 첨부파일 ID 없음 (Script Properties 미설정 또는 DriveApp 파일 찾기 실패) | 첨부파일 없이 발송 후 Logger 경고 |
| 체크된 행 없음 | "발송 대기 중인 항목이 없습니다." 알림 |
| N열 값이 드롭다운 목록에 없는 값 | 기본 템플릿으로 폴백 |

## 발송 구현 세부사항

```javascript
// GmailApp.sendEmail 옵션
GmailApp.sendEmail(recipientEmail, subject, '', {
  htmlBody: htmlBody,
  attachments: [DriveApp.getFileById(fileId).getAs(MimeType.PDF)],
  name: managerName  // 발신자 표시명
});
```

- `htmlBody` 사용으로 bold, 링크 등 HTML 렌더링
- 첨부파일: `DriveApp.getFileById(ATTACHMENT_FILE_ID)` 로 드라이브 파일 조회
- `from` 파라미터 생략 → 스크립트 실행 계정이 발신자로 자동 설정

## onEdit 동시성

`onEdit` 핸들러는 배경색 변경(하이라이트)만 수행하며 별도 잠금 불필요.
실제 이메일 발송은 커스텀 메뉴 버튼 클릭 시 수행되며, 이 시점에는 단일 사용자가 순차적으로 실행하므로 LockService 불필요.

## 파일 구조

| 파일 | 변경 | 내용 |
|------|------|------|
| `EmailService.gs` | 신규 | 템플릿 빌드(`buildEmailBody`), 발송(`sendEmailRow`), 일괄 발송(`sendPendingEmails`) |
| `Code.gs` | 수정 | `onEdit` 핸들러에 S열 체크박스 감지 + 하이라이트 추가, `onOpen`으로 커스텀 메뉴 등록 (신규 함수) |
| `Config.gs` | 수정 | `COL.EMAIL_SENT: 17` 추가 (S열 인덱스, 현재 빈칸으로 정의된 열) |
