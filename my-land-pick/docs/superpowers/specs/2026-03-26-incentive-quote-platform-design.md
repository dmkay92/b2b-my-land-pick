# 인센티브투어 견적 플랫폼 설계 문서

**작성일**: 2026-03-26
**프로젝트**: incentive-quote

---

## Context

여행사와 랜드사가 현재 이메일·엑셀 파일로 수동 처리하는 인센티브투어(맞춤 패키지) 견적 프로세스를 디지털화한다. 여러 여행사와 랜드사가 자유롭게 가입하는 B2B 플랫폼으로, 견적 요청 → 견적서 제출 → 협업 수정 → 최종 확정까지 전 과정을 한 곳에서 처리할 수 있게 한다.

## 사용자 역할

| 역할 | 권한 |
|------|------|
| **여행사** | 견적 요청 생성, 랜드사 견적서 비교·선택, 최종 확정 |
| **랜드사** | 담당 국가 견적 요청 조회, 견적서 제출·버전 업로드, 채팅 |
| **관리자** | 가입 승인, 랜드사 담당 국가 지정, 전체 현황 모니터링 |

---

## 전체 플로우

```
1. 여행사 견적 요청 생성
        ↓
2. 해당 국가 담당 랜드사 전체에 알림 발송
        ↓
3. 각 랜드사가 견적서(엑셀) 제출
        ↓
4. 여행사가 랜드사별 견적서 확인 → 1곳 선택
        ↓
5. 선택된 랜드사와 버전 관리 협업 수정 (채팅 병행)
        ↓
6. 여행사 최종 확정 → 완료
```

## 견적서 엑셀 구조

### 시트 1 — 일정표
스페인 7박 9일 일정표(MIKI TRAVEL) 디자인 기준:
- 파란색 헤더 (회사명 + 행사명)
- 컬럼: 여행일자 | 여행지역 | 교통편 | 시간 | 여행일정 | 식사
- Day별 일정 행 (흰색/연회색 교대)
- 호텔 정보 행은 별도 색상으로 구분 (파란색 계열)

### 시트 2 — 견적서
동일 디자인 스타일 적용:
- 컬럼: 항목 | 날짜 | 세부내역 | 가격 | 횟수 | 인원/수량 | 합계 | 기타
- 항목 카테고리: 호텔 / 차량 / 식사 / 입장료 / 가이드비용 / 기타
- 하단: 총합계 행 + 1인당 견적가 자동 계산

### 파일 처리 흐름
1. 플랫폼이 ExcelJS로 빈 템플릿 엑셀 생성 → 랜드사가 다운로드
2. 랜드사가 템플릿을 오프라인에서 작성 후 플랫폼에 업로드
3. 업로드된 파일 → Supabase Storage 저장
4. 여행사: 버전 클릭 → 해당 버전 엑셀 다운로드

## 견적서 버전 관리

- 랜드사가 수정된 견적서를 새 버전으로 업로드
- 버전 목록: v1, v2, v3… 클릭하면 해당 버전 단독 조회
- 버전 간 diff 뷰는 MVP에서 제외 (추후 추가 가능)
- 채팅으로 수정 요청 사항 커뮤니케이션

## DB 스키마

```sql
-- 사용자 (여행사/랜드사/관리자)
users (
  id, email, role (agency|landco|admin),
  company_name, status (pending|approved|rejected),
  country_codes text[],   -- 랜드사 담당 국가 목록
  created_at
)

-- 견적 요청
quote_requests (
  id, agency_id, event_name, destination_country, destination_city,
  depart_date, return_date,
  adults, children, infants, leaders,  -- 인원 합계는 계산
  hotel_grade (3|4|5),
  deadline, notes, status (open|in_progress|closed|finalized),
  created_at
)

-- 견적서 (버전 관리)
quotes (
  id, request_id, landco_id, version int,
  file_url, status (draft|submitted|selected|finalized),
  submitted_at
)

-- 선택 및 확정
-- selected_at: 여행사가 랜드사 선택한 시점
-- finalized_at: 협업 수정 후 최종 확정한 시점
quote_selections (
  request_id PK, selected_quote_id,
  selected_at, finalized_at
)

-- 채팅방 (견적별 × 랜드사별)
chat_rooms (
  id, request_id, agency_id, landco_id
)

-- 메시지
messages (
  id, room_id, sender_id, content, created_at
)

-- 알림
notifications (
  id, user_id, type, payload jsonb, read_at, created_at
)
```

## 주요 화면

### 여행사 (`/agency`)
- 대시보드: 견적 요청 목록 (상태별 필터: 진행중·완료·마감)
- 견적 요청 생성 폼
- 요청 상세: 랜드사별 견적서 카드 → 버전 히스토리 → 엑셀 다운로드
- 랜드사 선택 버튼 → 협업 수정 모드

### 랜드사 (`/landco`)
- 대시보드: 담당 국가 견적 요청 목록 (마감일 임박 순)
- 요청 상세: 견적서 엑셀 업로드, 버전 목록

### 관리자 (`/admin`)
- 가입 승인 대기 목록
- 랜드사별 담당 국가 지정
- 전체 견적 현황 모니터링

### 플로팅 채팅창
- 화면 우하단 고정, 페이지 이동 시에도 유지
- 채팅방 목록: `[행사명] — [랜드사명]` 형태
- 읽지 않은 메시지 뱃지
- 채팅방 클릭 → 인라인 확장 (같은 화면)
- 견적별로 채팅방 분리 (동일 랜드사라도 견적마다 별도 채팅)

## 알림 시스템

### 이메일 알림 (Resend)
| 이벤트 | 수신자 |
|--------|--------|
| 견적 요청 접수 | 해당 국가 담당 랜드사 전체 |
| 견적서 제출됨 | 여행사 |
| 랜드사 선택됨 | 선택된 랜드사 |
| 새 채팅 메시지 | 상대방 |
| 최종 확정됨 | 랜드사 |

### 인앱 알림
- 헤더 벨 아이콘 + 뱃지
- 알림 클릭 → 해당 견적 페이지 이동

---

## 견적 포맷 결정

**플랫폼 통일 포맷 강제** (랜드사 개별 템플릿 사용 안 함)
- 이유: 여행사의 랜드사 간 견적 비교 용이, 협업 수정 기능 구현 단순화
- 보완: 기타·메모 자유 입력 필드 제공

---

## 국가-랜드사 할당

- 관리자가 각 랜드사의 `country_codes` 배열을 직접 지정
- 여행사가 견적 요청 시 `destination_country` 기준으로 해당 랜드사 목록 조회
- 동일 국가에 여러 랜드사 할당 가능

## 검증 계획

1. `npm run dev` 로컬 실행 후 전체 플로우 수동 테스트
2. 회원가입(여행사·랜드사) → 관리자 승인 → 견적 요청 생성 → 견적서 제출 → 랜드사 선택 → 버전 업로드 → 채팅 → 최종 확정
3. 엑셀 다운로드 후 스페인 일정표 디자인과 비교
4. 두 개의 브라우저 창으로 실시간 채팅 동시 테스트
5. Resend 대시보드에서 이메일 수신 확인
6. 페이지 이동 시 플로팅 채팅창 유지 여부 확인
