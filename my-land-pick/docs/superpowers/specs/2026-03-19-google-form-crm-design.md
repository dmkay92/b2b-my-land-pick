# Google Form → Google Sheets CRM 자동화 설계

**작성일:** 2026-03-19
**상태:** 승인됨

---

## 개요

구글폼으로 상담 내용을 입력하면, Google Apps Script가 Gemini API를 통해 줄글을 분석하고, 미리 정의된 구글시트 CRM 포맷에 맞게 자동으로 행을 추가하는 자동화 시스템.

---

## 구성 요소

### 1. 구글폼

문항 3개:

| 순서 | 문항명 | 입력 타입 |
|------|--------|-----------|
| 1 | 담당자 | 드롭다운 (시트의 담당자 목록 탭과 동기화) |
| 2 | 거래처명 | 단답형 |
| 3 | 상담내용 | 장문형 (줄글) |

- 폼 제출 시 타임스탬프 자동 기록됨
- 담당자 드롭다운 선택지는 구글시트 `담당자 목록` 탭에서 관리. `syncManagerDropdown()` 함수로 동기화.

### 1-1. 담당자 목록 시트 탭

구글시트 내 별도 탭 `담당자 목록`:
- A1: 헤더 (`이름`)
- A2~: 담당자 이름 목록 (한 줄에 한 명)

담당자 추가/변경 시 이 탭만 수정 후 `syncManagerDropdown()` 실행.

---

### 2. 구글시트 컬럼 매핑

시트는 B3부터 데이터가 시작됨 (1~2행은 헤더/장식 행). 컬럼 순서:

| 컬럼명 | 소스 | 처리 방식 |
|--------|------|-----------|
| # | 자동 | `getLastRow() - 2` 로 순차 증가 계산 |
| 인바운드/아웃바운드 | — | 빈칸 (수동 입력) |
| 거래처명 | 폼 필드 2 | 직접 기입 |
| 담당자명 | — | 빈칸 (수동 입력) |
| 전화번호 | — | 빈칸 (수동 입력) |
| E-mail | — | 빈칸 (수동 입력) |
| 주소 | — | 빈칸 (수동 입력) |
| 대지역 | — | 빈칸 (수동 입력) |
| 소지역 | — | 빈칸 (수동 입력) |
| Category | — | 빈칸 (수동 입력) |
| 담당자 | 폼 필드 1 | 직접 기입 |
| 미팅/전화 일시 | 폼 타임스탬프 | `Utilities.formatDate(timestamp, "Asia/Seoul", "yyyy-MM-dd")` 형식으로 기입 |
| 월 발권량 | Gemini 분석 | 드롭다운 매칭 |
| GDS 사용 유무 | Gemini 분석 | 드롭다운 매칭 |
| 메인 에이전시 | Gemini 분석 | 드롭다운 매칭 |
| 메인 타겟 | Gemini 분석 | 드롭다운 매칭 |
| 비고 | Gemini 분석 | 나머지 유용한 정보 요약 |

**행 삽입 전략:**
- 거래처명 컬럼(D열)을 위에서 아래로 순회하여 처음으로 비어있는 행을 찾아 그 행에 기입
- 탐색 시작은 3행(첫 데이터 행)부터
- `#` 값은 `(찾은 행 번호) - 2` (1~2행이 헤더이므로)
- 빈 행이 중간에 있어도 첫 번째 빈 거래처명 행에 기입하므로 안전

**동시 제출 처리:**
- 동시에 여러 폼이 제출될 경우 `#` 중복 가능성이 있음 (Apps Script 비원자적 특성)
- `LockService.getScriptLock()`으로 스크립트 잠금 처리하여 경쟁 조건 방지
- `lock.waitLock(30000)` 사용 (30초 내 획득 못 하면 자동 throw → catch에서 로깅 후 종료)

### 3. 드롭다운 선택지

**월 발권량**
- 없음/모름
- 2천만원 이하
- 2천만원~5천만원
- 5천만원~1억원
- 1억원 이상
- 2억원 이상

**GDS 사용 유무**
- O
- X

**메인 에이전시**
- 하나투어
- 모두투어
- 인터파크
- 노랑풍선
- 온라인투어
- 마이리얼트립
- 자체 BSP
- 그 외

**메인 타겟**
- 상용
- 인디비

### 4. 데이터 흐름

```
구글폼 제출
    ↓
Apps Script onFormSubmit 트리거
    ↓
LockService로 스크립트 잠금
    ↓
폼 응답에서 타임스탬프, 거래처명, 담당자, 상담내용 추출
    ↓
Gemini Flash API 호출 (상담내용 분석 요청)
    ↓
응답에서 마크다운 코드펜스(```json ... ```) 제거 후 JSON 파싱
    ↓
드롭다운 유효성 검사 (선택지에 없으면 빈칸 처리)
    ↓
구글시트에 신규 행 추가 (B열부터)
    ↓
잠금 해제
```

### 5. Gemini 프롬프트 전략

**API 엔드포인트:**
```
https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key=API_KEY
```

**응답 파싱:**
- Gemini는 지시에도 불구하고 응답을 마크다운 코드펜스(` ```json ... ``` `)로 감싸는 경우가 있음
- JSON.parse() 전에 정규식으로 코드펜스 제거 처리 필수
- 파싱 성공 후 각 키가 없으면 빈 문자열(`""`)로 처리 (파싱 에러가 아님)

**프롬프트 템플릿 (스켈레톤):**
```
다음은 여행사/항공 B2B 영업 상담 내용입니다.
아래 항목을 분석하여 JSON 형식으로만 응답하세요. 마크다운 코드블록 없이 순수 JSON만 반환하세요.

상담내용:
{상담내용}

추출 항목:
1. 월 발권량: 다음 중 하나만 선택 ["없음/모름", "2천만원 이하", "2천만원~5천만원", "5천만원~1억원", "1억원 이상", "2억원 이상"]. 판단 불가 시 ""
2. GDS 사용 유무: 다음 중 하나만 선택 ["O", "X"]. 판단 불가 시 ""
3. 메인 에이전시: 다음 중 하나만 선택 ["하나투어", "모두투어", "인터파크", "노랑풍선", "온라인투어", "마이리얼트립", "자체 BSP", "그 외"]. 판단 불가 시 ""
4. 메인 타겟: 다음 중 하나만 선택 ["상용", "인디비"]. 판단 불가 시 ""
5. 비고: 담당자 연락처, 요청사항, 특이사항 등 나머지 유용한 정보를 2~3줄로 요약. 없으면 ""

응답 형식:
{"월 발권량": "", "GDS 사용 유무": "", "메인 에이전시": "", "메인 타겟": "", "비고": ""}
```

### 6. 기술 스택

| 항목 | 선택 |
|------|------|
| 자동화 런타임 | Google Apps Script |
| 트리거 | onFormSubmit (이벤트 기반) |
| AI 모델 | Gemini Flash (`gemini-2.0-flash`) |
| AI 연동 방식 | `UrlFetchApp`으로 Gemini REST API 직접 호출 |
| 시트 조작 | `SpreadsheetApp` |
| 설정 저장 | `PropertiesService.getScriptProperties()` |

**Script Properties에 저장할 값:**
- `GEMINI_API_KEY`: Google AI Studio에서 발급한 API 키
- `SPREADSHEET_ID`: 대상 구글시트 ID (URL의 `/d/` 뒤 문자열)
- `FORM_ID`: 구글폼 ID (폼 URL의 `/d/` 뒤 문자열). `syncManagerDropdown()`에서 사용.

---

### 7. 에러 처리

모든 에러 케이스에서 거래처명과 타임스탬프는 반드시 기입하여 데이터 유실 방지.

| 에러 상황 | 처리 방식 |
|-----------|-----------|
| Gemini API 호출 실패 | 거래처명 + 타임스탬프 기입, 비고에 "AI 분석 실패 - 원문: {상담내용}" 기입 |
| JSON 파싱 실패 | 거래처명 + 타임스탬프 기입, 비고에 원문 상담내용 기입 |
| 드롭다운 값 불일치 | 해당 컬럼 빈칸 처리 |
| Gemini 응답에 키 누락 | 해당 키를 빈 문자열로 처리 (에러 아님) |
| LockService 잠금 획득 실패 | `Logger.log()` 후 `throw` (실행 로그에 기록, 데이터 미기입) |

---

### 8. 로깅

- `Logger.log()`를 API 호출 전후, 시트 기입 후 주요 단계마다 기록
- Apps Script 편집기의 실행 로그에서 디버깅 가능

---

### 9. 설정 방법 (요약)

1. Google AI Studio에서 Gemini API 키 발급
2. Apps Script 편집기 → 프로젝트 설정 → 스크립트 속성에 `GEMINI_API_KEY`, `SPREADSHEET_ID` 저장
3. 구글폼에 스크립트 연결 후 `onFormSubmit` 트리거 등록 (이벤트 기반)
4. 테스트 제출로 동작 확인
