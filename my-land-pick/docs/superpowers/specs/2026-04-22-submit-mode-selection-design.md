# 견적 제출 시 모드 선택 기능

## 배경

랜드사가 v1에서 항목별로 작성한 뒤 v2에서 합계만으로 제출하고 싶을 때, 에디터에 이전 항목별 데이터가 남아있어 실수로 잘못된 모드/금액으로 제출될 위험이 있다. 제출 시점에 명시적으로 모드를 선택하게 하여 이를 방지한다.

## 변경 범위

### 1. 제출 모달 추가

제출 버튼 클릭 시 바로 제출하지 않고, 모달에서 "합계만 / 항목별" 선택 후 제출한다.

```
[제출하기] 클릭
    ↓
┌─────────────────────────┐
│  견적 제출 방식 선택       │
│                         │
│  ○ 합계만 (총액만 전달)    │
│  ○ 항목별 (상세 내역 포함)  │
│                         │
│  ⚠ 합계 금액이 입력되지    │
│    않았습니다. (조건부)     │
│                         │
│  [취소]     [제출하기]     │
└─────────────────────────┘
```

- 모달의 기본 선택값: 현재 에디터의 `pricingMode` 상태
- "합계만" 선택 시 `summary_total`이 비어있으면 경고 메시지 표시 + 제출 버튼 비활성화
- "항목별" 선택 시 pricing 항목 데이터가 비어있으면 경고 메시지 표시 + 제출 버튼 비활성화

### 2. 에디터는 변경 없음

- 기존 항목별/합계만 토글 유지
- 자유롭게 왔다갔다 가능 (v3에서 다시 항목별로 수정 제출 가능)

### 3. 제출 API 변경

`POST /api/quotes/draft/submit` 호출 시 모달에서 선택한 `pricing_mode`를 전달한다.

- 기존: 드래프트에 저장된 `pricing_mode`를 그대로 사용
- 변경: 요청 body의 `pricing_mode`를 최종값으로 사용하여 DB에 저장

### 4. 하류 영향 없음

견적 카드, 엑셀 다운로드는 기존대로 DB의 `pricing_mode` 기준으로 동작한다. 모달에서 선택한 값이 DB에 정확히 들어가므로 추가 변경 불필요.

## 수정 대상 파일

| 파일 | 변경 내용 |
|------|----------|
| `src/app/(dashboard)/landco/requests/[id]/quote/new/page.tsx` | 제출 버튼 클릭 시 모달 열기 |
| 새 컴포넌트: `src/components/SubmitModeModal.tsx` | 모드 선택 모달 UI + 유효성 검증 |
| `src/app/api/quotes/draft/submit/route.ts` | body에서 `pricing_mode` 받아 최종값으로 사용 |
