# macOS 캘린더 앱 디자인 문서

**작성일:** 2026-02-11
**프로젝트:** macOS 스탠드얼론 캘린더 애플리케이션
**대상 사용자:** 비개발자도 유지보수 가능한 간단한 구조

---

## 프로젝트 개요

### 목적
macOS 전용 데스크톱 캘린더 애플리케이션으로, 자체 데이터베이스를 사용하여 이벤트를 관리하고 알림을 제공합니다.

### 주요 기능
- ✅ 월간/주간/일간 캘린더 뷰
- ✅ 이벤트 생성/수정/삭제
- ✅ 반복 이벤트 지원
- ✅ 데스크톱 알림/리마인더
- ✅ 간단한 검색 기능
- ✅ 라이트/다크 모드

### 기술 스택
- **Electron 28+**: 데스크톱 앱 프레임워크
- **Toast UI Calendar 2.x**: 캘린더 UI 라이브러리
- **better-sqlite3**: 로컬 데이터베이스
- **순수 HTML/CSS/JavaScript**: 복잡한 프레임워크 없음

## 1. 아키텍처 설계

### 레이어 구조

#### 1.1 프레젠테이션 레이어 (UI)
- HTML 파일들로 메인 윈도우 구성
- CSS로 스타일링 (라이트/다크 테마)
- JavaScript로 UI 인터랙션 처리
- Toast UI Calendar 라이브러리가 캘린더 렌더링 담당

#### 1.2 애플리케이션 레이어 (비즈니스 로직)
- **이벤트 매니저**: 생성/수정/삭제 작업 처리
- **알림 매니저**: 리마인더 스케줄링 및 트리거
- **검색 매니저**: 제목으로 이벤트 검색
- **반복 엔진**: 반복 이벤트 인스턴스 계산

#### 1.3 데이터 레이어 (저장소)
- SQLite 데이터베이스, 두 개의 테이블:
  - `events`: 모든 이벤트 데이터 저장
  - `settings`: 사용자 설정 저장

### 메인 윈도우 레이아웃

```
┌─────────────────────────────────────┐
│  [< > 오늘] [월|주|일] [🔍]          │  ← 헤더
├─────────────────────────────────────┤
│                                     │
│         캘린더 뷰 영역               │  ← 메인 콘텐츠
│      (Toast UI Calendar)            │
│                                     │
├─────────────────────────────────────┤
│  [+ 새 이벤트]                       │  ← 푸터
└─────────────────────────────────────┘
```

## 2. 데이터 모델

### 2.1 events 테이블

```sql
CREATE TABLE events (
  id TEXT PRIMARY KEY,              -- 고유 ID (UUID)
  title TEXT NOT NULL,              -- 이벤트 제목
  description TEXT,                 -- 설명
  start_date TEXT NOT NULL,         -- 시작 날짜/시간 (ISO 8601)
  end_date TEXT NOT NULL,           -- 종료 날짜/시간
  location TEXT,                    -- 장소
  category TEXT,                    -- 카테고리 이름
  color TEXT,                       -- 색상 코드 (#FF5733)

  -- 반복 이벤트 관련
  is_recurring INTEGER DEFAULT 0,   -- 반복 여부 (0/1)
  recurrence_rule TEXT,             -- 반복 규칙 (JSON)
  parent_event_id TEXT,             -- 부모 이벤트 ID

  -- 알림 관련
  reminder_minutes INTEGER,         -- 몇 분 전 알림 (15, 30, 60 등)

  created_at TEXT NOT NULL,         -- 생성 시간
  updated_at TEXT NOT NULL          -- 수정 시간
);
```

### 2.2 settings 테이블

```sql
CREATE TABLE settings (
  key TEXT PRIMARY KEY,             -- 설정 키
  value TEXT NOT NULL               -- 설정 값 (JSON)
);
```

### 2.3 반복 규칙 예시

```json
{
  "frequency": "daily|weekly|monthly|yearly",
  "interval": 1,
  "daysOfWeek": [0, 1, 2, 3, 4],
  "endDate": "2026-12-31",
  "occurrences": 10
}
```

### 2.4 기본 카테고리

- **개인** (파란색 #4285F4)
- **업무** (빨간색 #EA4335)
- **가족** (초록색 #34A853)
- **기타** (회색 #9E9E9E)

사용자가 커스텀 카테고리를 추가할 수 있습니다.

## 3. UI 컴포넌트

### 3.1 헤더 바
- 네비게이션 버튼: `[◀ 이전] [오늘] [다음 ▶]`
- 뷰 전환 탭: `[월간] [주간] [일간]`
- 검색 아이콘 🔍 클릭 시 검색 바 토글
- 설정 아이콘 ⚙️ (테마, 기본 뷰 등)

### 3.2 캘린더 영역
- Toast UI Calendar가 렌더링
- **월간 뷰**: 그리드 형식, 각 날짜 셀에 이벤트 표시
- **주간 뷰**: 시간대별 세로 레이아웃 (00:00-23:59)
- **일간 뷰**: 선택된 날짜의 시간대별 상세 뷰
- 이벤트 클릭 시 상세 정보 팝업

### 3.3 이벤트 생성/수정 모달

```
┌─────────────────────────────┐
│  새 이벤트               [X] │
├─────────────────────────────┤
│  제목: [____________]        │
│  시작: [날짜] [시간]         │
│  종료: [날짜] [시간]         │
│  설명: [____________]        │
│  장소: [____________]        │
│  카테고리: [▼ 선택]          │
│  색상: [🔴🟢🔵⚪]         │
│  반복: [▼ 없음/매일/...]     │
│  알림: [▼ 15분 전]          │
├─────────────────────────────┤
│        [취소]  [저장]        │
└─────────────────────────────┘
```

### 3.4 검색 인터페이스
- 헤더 아래 슬라이드 다운
- 실시간 검색 (타이핑하면 즉시 필터링)
- 결과를 날짜순으로 리스트 표시
- 클릭하면 해당 날짜로 이동 및 이벤트 하이라이트

### 3.5 설정 패널
- 테마: 라이트/다크/자동
- 기본 뷰: 월간/주간/일간
- 주 시작일: 일요일/월요일
- 알림 기본값: 15분/30분/1시간 전

## 4. 이벤트 관리

### 4.1 이벤트 생성 플로우

1. **트리거 방법**
   - 하단 `[+ 새 이벤트]` 버튼 클릭
   - 캘린더에서 날짜/시간 셀 더블클릭
   - 단축키: `Cmd + N`

2. **모달 열림**
   - 선택된 날짜/시간으로 자동 채워짐
   - 모든 필드 입력 (제목은 필수)
   - 반복 설정 시 반복 옵션 패널 확장

3. **저장 처리**
   - 유효성 검사 (제목 필수, 종료 시간 > 시작 시간)
   - SQLite에 저장
   - 알림이 설정되어 있으면 알림 스케줄 등록
   - 캘린더 뷰 자동 업데이트

### 4.2 이벤트 수정 플로우

1. 캘린더에서 이벤트 클릭
2. 상세 정보 팝업 표시 (읽기 전용)
3. `[수정]` 클릭 시 편집 모달로 전환
4. 반복 이벤트의 경우 선택 옵션:
   - "이 이벤트만"
   - "이후 모든 이벤트"
   - "모든 반복 이벤트"

### 4.3 이벤트 삭제 플로우

1. 상세 팝업에서 `[삭제]` 클릭
2. 확인 다이얼로그: "정말 삭제하시겠습니까?"
3. 반복 이벤트인 경우 동일한 선택 옵션 제공
4. 삭제 후 알림 스케줄도 함께 제거
5. 캘린더 뷰 자동 업데이트

### 4.4 빠른 편집
- 캘린더에서 이벤트를 드래그하면 날짜/시간 변경
- 이벤트 끝부분을 드래그하면 시간 연장/단축

## 5. 반복 이벤트 처리

### 5.1 반복 옵션

- 없음 (기본값)
- 매일
- 매주 (요일 선택 가능)
- 매월 (날짜 기준)
- 매년
- 커스텀...

### 5.2 커스텀 반복 설정

```
┌─────────────────────────────┐
│  반복 설정                  │
├─────────────────────────────┤
│  간격: [1] [▼ 주]마다       │
│  반복 요일:                 │
│  [✓월] [✓화] [✓수] [✓목] [✓금] │
│  [ 토] [ 일]                │
│                             │
│  종료 조건:                 │
│  ⦿ 종료일: [2026-12-31]     │
│  ○ 횟수: [10]회 후          │
│  ○ 종료 안 함               │
├─────────────────────────────┤
│        [취소]  [확인]       │
└─────────────────────────────┘
```

### 5.3 저장 방식 (Virtual Instances)

- 반복 이벤트는 부모 이벤트 1개만 DB에 저장
- `recurrence_rule`에 반복 규칙 저장
- 캘린더 렌더링 시 규칙을 기반으로 인스턴스 동적 생성
- **장점**: DB 용량 절약, 규칙 수정 시 한 번만 업데이트

### 5.4 예외 처리

- 특정 인스턴스를 수정하면 새 이벤트로 생성
- `parent_event_id`로 부모와 연결
- 부모 이벤트 렌더링 시 해당 날짜는 제외하고 예외 이벤트 표시

### 5.5 삭제 로직

- **이 이벤트만**: 예외로 표시 (숨김 플래그)
- **이후 모든 이벤트**: 부모의 `endDate` 수정
- **모든 반복 이벤트**: 부모 및 모든 예외 삭제

## 6. 알림/리마인더 시스템

### 6.1 알림 스케줄링

- 앱 시작 시 다음 7일간의 모든 이벤트 알림을 메모리에 로드
- 매일 자동으로 다음 날 알림 추가 (백그라운드 태스크)
- 반복 이벤트의 경우 각 인스턴스마다 알림 생성

### 6.2 알림 트리거

```
알림 시간 = 이벤트 시작 시간 - reminder_minutes
```

### 6.3 알림 표시

- macOS 네이티브 알림 사용
- 알림 내용:
  ```
  🗓️ [카테고리] 제목
  ⏰ 15분 후 시작 (14:00-15:00)
  📍 장소 (있는 경우)
  ```
- 알림 클릭 시 앱 활성화 및 해당 이벤트로 이동

### 6.4 알림 옵션

- 알림 없음
- 이벤트 시작 시
- 5분 전
- 15분 전 (기본값)
- 30분 전
- 1시간 전
- 1일 전

### 6.5 백그라운드 실행

- 앱이 닫혀 있어도 알림 받기 위해:
  - macOS 트레이(메뉴바)에 아이콘 유지
  - "완전 종료" 옵션 제공
  - 시스템 시작 시 자동 실행 옵션

### 6.6 스누즈 기능

- 알림이 뜨면 "다시 알림" 버튼
- 5분/10분 후 재알림 선택 가능

## 7. 검색 기능

### 7.1 검색 활성화

- 헤더의 🔍 아이콘 클릭
- 단축키: `Cmd + F`
- 검색창이 헤더 아래로 슬라이드 다운

### 7.2 검색 로직

**실시간 검색**
- 사용자가 타이핑할 때마다 즉시 검색
- 300ms 디바운싱 적용 (성능 최적화)

**검색 범위**
- 이벤트 제목 (주 검색 대상)
- 설명 (부 검색 대상)
- 장소 (부 검색 대상)

**검색 쿼리**
```sql
SELECT * FROM events
WHERE title LIKE '%검색어%'
   OR description LIKE '%검색어%'
   OR location LIKE '%검색어%'
ORDER BY start_date ASC
LIMIT 50;
```

### 7.3 검색 결과 표시

- 날짜/시간 기준 오름차순 정렬
- 과거 이벤트도 포함
- 각 결과 클릭 시:
  - 검색창 닫기
  - 해당 날짜로 캘린더 이동
  - 이벤트 하이라이트 (노란색 테두리 2초간)
  - 이벤트 상세 팝업 자동 열기

### 7.4 향후 확장 가능

- 현재는 간단한 텍스트 검색만
- 나중에 필요하면 추가 가능:
  - 날짜 범위 필터
  - 카테고리 필터
  - 고급 검색

## 8. 테마 시스템 (라이트/다크 모드)

- 라이트 모드
- 다크 모드
- 자동 (시스템 설정 따라가기)

### 8.2 색상 팔레트

**라이트 모드**
```
배경: #FFFFFF
카드 배경: #F5F5F5
텍스트: #212121
보조 텍스트: #757575
테두리: #E0E0E0
강조색: #4285F4
```

**다크 모드**
```
배경: #1E1E1E
카드 배경: #2D2D2D
텍스트: #FFFFFF
보조 텍스트: #B0B0B0
테두리: #404040
강조색: #8AB4F8
```

### 8.3 구현 방식

**CSS Variables 사용**
```css
:root {
  --bg-primary: #FFFFFF;
  --text-primary: #212121;
}

[data-theme="dark"] {
  --bg-primary: #1E1E1E;
  --text-primary: #FFFFFF;
}
```

### 8.4 자동 테마 전환

```javascript
// 시스템 테마 변경 감지
window.matchMedia('(prefers-color-scheme: dark)')
  .addEventListener('change', (e) => {
    if (settings.theme === 'auto') {
      applyTheme(e.matches ? 'dark' : 'light');
    }
  });
```
